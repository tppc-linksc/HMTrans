# v0.3 产品与交互方案

本文定义 `v0.3.0` 的产品和交互基线。稳定版文件互传保持现状，本版本只新增 MatePad 到 Mac 的单向画面投放；声音和反向控制不属于首版。

## 1. 使用目标

核心场景是用户在 MatePad 上演示应用，同时在 Mac 上观看并使用 macOS 系统录屏保存演示视频。HMTrans 只负责实时显示 Pad 画面，不把 Pad 伪装为副屏，也不在应用内保存录像。

### 成功标准

- 已配对设备能在 3 秒内进入系统录屏授权或明确错误状态。
- 授权后用户可以切换到其他 Pad 应用，Mac 继续显示完整屏幕画面。
- 横竖屏切换后画面方向、宽高比和观看窗口正确更新。
- 局域网短暂抖动后自动恢复，不能一直停留在“连接中”。
- 停止后立即结束采集、编码、网络和解码，不在后台继续录屏。

## 2. 信息架构

双端继续使用三个一级入口：

| 入口 | 现有能力 | v0.3 新增 |
| --- | --- | --- |
| 连接 | 发现、配对、选择设备、发送文件/文件夹 | 对已连接 Mac 发起投屏；显示投屏中的状态和停止入口 |
| 历史 | 文件任务与历史 | 不记录视频帧；可选记录不含画面的本地诊断事件 |
| 设置 | 服务、目录、网络、诊断和隐私 | 投屏接收开关、画质策略、投屏端口和隐私说明 |

Mac 观看画面使用独立窗口，不挤入连接页，并支持切换为画中画小窗。关闭主窗口不终止正在进行的投屏；关闭观看窗口即停止当前投屏，避免窗口消失后仍在后台接收敏感画面。

## 3. Pad 发起流程

1. 连接页只对“已配对且双向在线”的 Mac 显示投屏操作。
2. 已连接 Mac 设备卡片右侧显示屏幕图标和“投屏”文字。卡片主体继续用于选择文件发送目标，投屏操作使用独立点击区域，不能让点选设备意外开始录屏；长按菜单只作为补充入口。
3. 用户点按“投屏”。未连接设备先完成连接，未配对设备先输入对方六位码。
4. HMTrans 连接 Mac 的独立投屏端口。Mac 关闭投屏接收、配对身份不匹配或正被占用时，Pad 直接显示原因。
5. Mac 自动校验已配对身份并接受后，Pad 调用 HarmonyOS 系统屏幕采集授权。配对不能替代这一步，也不能自动点击系统授权。
6. 用户允许后进入“正在投屏”，并可以切换到其他应用。用户拒绝时回到连接页，不自动重试。
7. 用户从系统录屏状态入口、HMTrans 连接页或 Mac 观看窗口停止会话。

首版不允许从 Mac 远程要求 Pad 无提示开始录屏，也不提供 Mac 主动拉起 Pad 投屏页的命令；最终发起和授权必须发生在 Pad。

## 4. Pad 连接页状态

设备卡片在现有连接状态之外增加投屏状态文字：

| 状态 | 文案 | 可用操作 |
| --- | --- | --- |
| 可投屏 | 已连接 | 投屏到此 Mac、发送文件 |
| 邀请中 | 正在请求 Mac 接收 | 取消 |
| 等待授权 | 等待系统录屏授权 | 取消 |
| 启动中 | 正在建立画面 | 取消 |
| 投屏中 | 正在投屏 · 1080p / 30 fps | 停止投屏、发送文件 |
| 重连中 | 网络波动，正在重连 | 立即停止 |
| 不可用 | 对方未开启投屏接收 | 打开帮助 |

投屏中的设备卡片不能因为普通 UDP 心跳短暂抖动立即显示离线；实时会话心跳优先决定状态。超过重连窗口仍未恢复后才回到离线/已配对状态。

## 5. Pad 会话提示

- 系统屏幕采集标识和系统停止入口始终保留，应用不得伪装、遮挡或绕过。
- 回到 HMTrans 时，连接页顶部显示精简会话条：目标 Mac、分辨率、连接质量、持续时间和停止按钮。
- 没有正在投屏时不显示空的投屏卡片。
- Pad 锁屏、系统终止录屏或设备进入不允许采集的安全页面时，应用按系统结果停止或显示受保护内容，不使用截图等替代手段绕过限制。

## 6. Mac 接收与观看

### 6.1 接收与身份校验

- 只接受已配对、身份指纹匹配、当前在线的 Pad。
- “投屏接收”服务开启时，已配对且身份密钥匹配的 Pad 会自动通过接收校验并打开观看窗口；未配对设备和旧版无投屏密钥的配对记录一律拒绝。
- 同一 Mac 已有投屏时，第二个邀请返回“Mac 正在显示其他设备”，不静默替换。
- Mac 空闲时不在连接页放置“开始投屏”按钮，因为 Mac 不能绕过 Pad 主动发起和系统授权。活动会话对应的 Pad 卡片右侧显示“打开画面”，用于找回已隐藏的普通窗口或画中画。

### 6.2 观看窗口

窗口标题显示设备名和会话状态。工具栏保持最少操作：

- 适应窗口 / 原始比例。
- 切换普通窗口 / 画中画。
- 进入或退出全屏。
- 显示或隐藏画面信息（分辨率、帧率、码率、延迟和丢帧数）。
- 停止投屏。

画面默认保持宽高比并使用黑色或系统背景补边，不拉伸、不裁剪。方向改变时平滑更新窗口内容区域；除非用户手动进入全屏，不强制抢占焦点或改变用户设置的窗口位置。

### 6.3 画中画模式

画中画属于 `v0.3.0` 首版能力，主要用于录制 HMTrans 演示：用户可以在 Mac 上保持主窗口可操作，同时把 Pad 画面放在屏幕角落。

- 由观看窗口工具栏的“画中画”按钮进入，也可以从活动设备卡片的“打开画面”菜单直接选择。
- 使用独立的 Mac 小型置顶窗口实现，不依赖 Pad 系统画中画，也不改变 Pad 上被采集应用的窗口形态。
- 小窗保持画面宽高比，可拖动、缩放并吸附到屏幕四角；建议最小内容尺寸约 `320 × 180`，竖屏时按实际比例调整。
- 进入画中画后隐藏厚重标题栏，鼠标移入时才显示“恢复普通窗口”和“停止投屏”等最少控制。
- 普通窗口、画中画和全屏共享同一个解码会话，切换模式不能重连、重复解码或出现黑屏。
- 记住最后位置和大小，但显示器变化或窗口越界时自动移回当前可见区域。
- 使用 macOS 录制整个显示器时可以同时录入主窗口和画中画；如果用户只录制某个单独窗口，是否包含画中画由 macOS 的录制目标决定，HMTrans 不做画面合成。

### 6.4 窗口生命周期

- 主窗口关闭不影响观看窗口。
- 普通窗口与画中画是同一会话的两种外观，任意时刻只显示一个，不能生成多个重复观看窗口。
- 用户关闭当前观看窗口即停止会话；从连接页可以在活动会话期间重新打开普通窗口或画中画。
- Mac 应用退出、接收开关关闭、设备睡眠或用户停止时，向 Pad 发送结束原因并释放解码器。
- 睡眠前停止会话；唤醒后不自动重新采集 Pad 屏幕，用户需要再次发起并授权。

## 7. 设置项

### Pad

- **默认画质**：流畅、均衡、高清。默认“均衡”；不同档位分别调整目标尺寸、帧率和码率。
- **投屏端口**：独立 TCP 端口，默认 `51890`，必须与 Mac 保持一致。
- **投屏诊断信息**：复制不含画面、文件名、IP、配对码或身份原值的本地状态。

### Mac

- **投屏接收**：关闭后拒绝新会话，并确认是否停止当前会话。
- **投屏端口**：使用单独的“投屏服务”编辑入口，默认 `51890`；不与文件发现和传输端口绑定重启。
- **观看模式**：普通窗口不强制置顶；进入画中画后使用原生浮动小窗。

所有设置必须持久化并真实生效。修改投屏端口时只重启投屏接收服务；修改文件网络端口不影响投屏会话，失败时显示明确原因。

## 8. 文件传输共存

- 投屏中仍允许发送小文件和截图，不要求用户先停止投屏。
- 投屏与文件传输使用独立端口、连接、队列和任务状态，任一链路失败不能污染另一链路的任务 ID、断点或校验结果。
- 首版不对文件传输动态限速；带宽竞争由真机验收测量，必要时用户可暂停文件任务，后续版本再根据数据增加自适应调度。
- 历史页只显示文件任务；投屏会话不生成“已完成文件”记录。

## 9. 错误与恢复

所有错误必须包含“发生了什么”和“下一步怎么做”：

| 场景 | 用户提示 | 行为 |
| --- | --- | --- |
| 系统授权拒绝 | 未允许屏幕录制，可再次发起 | 不重试、不启动编码 |
| Mac 未开启接收 | 对方未开启投屏接收 | 保持设备连接，可继续传文件 |
| 编码器不可用 | 当前设备无法启动 H.264 编码 | 释放采集资源并记录诊断 |
| 首帧超时 | 未收到画面，正在重试 | 请求关键帧；超限后结束 |
| 网络抖动 | 网络波动，正在重连 | 10 秒内有限重试，成功后请求关键帧 |
| 身份变化 | 对方身份已变化，请重新配对 | 终止会话并撤销旧信任 |
| Mac 解码失败 | 画面解码失败，请重新投屏 | 清空解码队列，不影响文件服务 |

错误界面不能显示原始系统对象、空的 `{}` 或无限旋转状态。

## 10. 隐私与明确边界

- 投屏会包含用户屏幕上可见的应用、通知和个人信息；首次使用前在应用内说明，最终授权以 HarmonyOS 系统弹窗为准。
- HMTrans 不保存、截图、索引、上传或分析画面；Mac 只在内存中解码显示。
- 用户若使用 macOS 系统录屏保存视频，该文件由 macOS 与用户选择的目录管理，不属于 HMTrans 自动收集。
- 诊断只记录时间、阶段、错误码、编码参数和性能统计，不记录帧内容。
- `v0.3.0` 不采集或传输声音，不监听键盘鼠标，不注入触控事件。

## 11. 平台与可访问性

- Pad 保持原生沉浸光感 Dock 和深浅色资源；投屏操作使用点按/长按和系统确认，不依赖悬浮或右键。
- Mac 使用独立原生窗口、画中画、系统全屏和键盘快捷键；关键操作支持 VoiceOver。
- 窄窗口、Pad 竖屏、分屏和浮窗下，投屏状态条可以换行但停止按钮始终可见。
- 状态不能只靠颜色表达，必须同时有文字或图标。

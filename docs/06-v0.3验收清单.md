# v0.3 验收清单

本清单用于 `v0.3.0` 投屏发布候选。2026-07-22 用户已确认当前目标设备核心投屏流程测试通过，但没有对应日志、截图或性能记录的细分条目仍保持未勾选；构建、安装或单次成功不能代替长时、极端网络和压力场景证据。

## 1. 版本与安装

- [ ] Mac 与 Pad 对外 `versionName` 一致，内部构建号分别递增。
- [ ] 从 `v0.2.1` 覆盖升级后，文件历史、设置、安装身份和配对关系保留。
- [ ] 完整卸载/清除应用数据后生成新身份，双方旧信任失效并重新输入一次六位码。
- [ ] Mac DMG 干净安装、首次打开和安全提示说明与实际 ad-hoc 签名一致。
- [ ] HarmonyOS 正式候选 HAP 干净安装、隐私同意、目录授权和投屏系统授权顺序正确。

## 2. 基础投屏流程

- [ ] 只有已配对且在线的 Mac 显示“投屏到此 Mac”。
- [ ] Mac 主窗口不显示主动投屏入口，只保留接收服务和独立观看窗口。
- [ ] 从 Pad 发起后在 3 秒内出现系统屏幕采集确认；用户确认前不采集、不编码、不发送画面。
- [ ] 系统拒绝授权时 Pad 显示明确结果，Mac 不假显示“正在投屏”。
- [ ] Mac 关闭接收、忙碌或版本不支持时，Pad 显示明确原因。
- [ ] 用户允许系统录屏后 Mac 显示第一帧；拒绝后不启动采集和编码。
- [ ] 用户切换到其他 Pad 应用后 Mac 继续显示系统允许的画面。
- [ ] Pad 页面切换、倒计时和连续动画在 Mac 上持续更新，不停在首帧或只更新统计数字。
- [ ] Pad 和 Mac 任一端停止后，双方在 1 秒内退出会话状态并释放资源。
- [ ] 投屏不创建文件历史、不生成视频文件、不污染下载目录。

## 3. 显示与窗口

- [ ] Pad 横屏、竖屏和多次方向切换后方向与比例正确。
- [ ] Mac 适应窗口、原始比例、缩放和全屏不拉伸、不裁剪。
- [ ] 普通窗口、画中画和全屏切换不重连、不重复解码、不黑屏。
- [ ] 画中画保持比例、可拖动缩放、置顶和四角放置，显示器变化后不会留在屏幕外。
- [ ] 录制整个 Mac 显示器时，HMTrans 主窗口与 Pad 画中画可以同时出现在录制画面中。
- [ ] 主窗口关闭后观看窗口按设计继续；应用退出时会话结束。
- [ ] 深浅色、窄窗口、不同 Retina 缩放和多显示器行为正确。
- [ ] 关闭观看窗口会直接停止会话，不会留下后台录屏或残留接收状态。

## 4. 性能目标

- [ ] 接受邀请到进入系统授权/明确错误不超过 3 秒。
- [ ] 授权完成到第一帧在典型局域网下不超过 3 秒。
- [ ] 流畅、均衡和画质三个档位分别连续运行；需要降低负载时由用户切换档位，状态与实际编码参数一致。
- [ ] 均衡/高清以 60 fps 为请求上限；实际达不到时如实显示采集、发送和解码帧率，不以网络包数冒充显示帧率。
- [ ] 典型局域网玻璃到玻璃延迟：P50 不高于 250 ms，P95 不高于 450 ms。
- [ ] 方向切换后 2 秒内恢复正确关键帧画面。
- [ ] Pad 待发送帧不超过 4 帧/32 MiB，Mac 解码队列不超过 4 帧。
- [ ] 内存达到稳态后没有持续线性增长；CPU、温度和功耗已记录机型与基线。

## 5. 网络与恢复

- [ ] Wi-Fi 短暂抖动进入“重连中”，10 秒内恢复后请求新关键帧。
- [ ] 超过重连窗口后双方停止，不无限重试或假显示“已连接”。
- [ ] Mac 睡眠前结束会话，唤醒后不绕过 Pad 授权自动恢复录屏。
- [ ] Pad 锁屏、系统撤销录屏或应用被终止时 Mac 收到结束/超时并退出。
- [ ] 路由器切换、IP 变化和重复发现不会生成同一安装身份的重复设备。
- [ ] 半开连接、畸形帧和连接风暴不会造成 OOM、UI 卡死或接收服务失效。

## 6. 文件传输共存与回归

- [ ] 投屏中 Pad→Mac 和 Mac→Pad 小文件仍可完成。
- [ ] 投屏与 5 GB/20 GB+ 文件并行时，投屏保持可用，文件任务可暂停/恢复且 SHA-256 正确。
- [ ] 投屏停止不改变文件任务的并发配置、状态或断点，不创建重复任务。
- [ ] 文件夹、深层嵌套、断点续传、一对多不同文件和解除配对全部回归通过。
- [ ] A-B、B-C 已配对而 A-C 未配对时，画面和文件都不会经 B 中继。

## 7. 身份、安全与隐私

- [ ] 未配对、身份变化、错误目标、错误共享密钥和重复序号全部被拒绝。
- [ ] 正式候选的媒体与敏感控制载荷实际加密，并通过完整性和重放测试。
- [ ] Mac 校验 HMCS 会话的安装身份、目标设备、共享密钥和连接内序号；伪造与重放数据被拒绝。
- [ ] 六位码不作为长期媒体密钥；会话结束后会话密码对象与帧缓冲从内存释放，配对共享密钥只保存在本地并在解除配对时删除。
- [ ] Mac 不落盘、不截图、不索引画面；诊断中没有画面、参数集、密钥、完整 IP 和完整路径。
- [ ] 首次投屏说明、系统授权、隐私政策和应用市场权限声明一致。
- [ ] 系统受保护页面按平台规则处理，不使用替代手段绕过限制。

## 8. 交互与可访问性

- [ ] Pad 点按、长按、停止和错误恢复符合触摸操作，不依赖悬浮或右键。
- [ ] Mac 接收窗口、画中画、全屏、关闭、停止和键盘焦点符合原生窗口行为。
- [ ] 状态不只依赖颜色；关键按钮具备可读标签和足够触控区域。
- [ ] VoiceOver/系统读屏能读出设备、会话状态和停止操作。
- [ ] Pad 全屏、横竖屏、分屏和浮窗逐页截图通过，无 Dock/系统手势区冲突。

## 9. 长期与重复操作

- [ ] 100 次发起/停止无崩溃、无残留录屏标识、无端口占用。
- [ ] 100 次拒绝授权、断网、窗口关闭和应用切换无状态错乱。
- [ ] 空闲 30 分钟无采集器、编码器、解码器、高频 UI 刷新或后台长时任务。
- [ ] 投屏 60 分钟后停止，内存、线程、文件描述符和网络连接回到合理基线。

## 10. 证据要求

每个关键条目至少记录：

- 双端版本、构建号、设备型号和系统版本。
- 网络环境和测试文件/画面场景。
- 操作步骤、结果、日志或截图/录屏。
- 首帧、延迟、帧率、码率、CPU、内存和温度数据（适用时）。
- 失败条目的错误码、诊断信息和对应 issue。

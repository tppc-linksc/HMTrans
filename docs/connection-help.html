应用市场版首次打开后，可能无法发现 Mac

如果 MatePad 和 Mac 已连接同一个 Wi-Fi，但两端一直显示未连接，这通常不是你的网络配置错误，而是当前 HarmonyOS v0.1 应用市场安装包的首次启动问题。

问题确认时间：2026 年 7 月 10 日 · 影响范围：HarmonyOS 应用市场版 v0.1

现在可以先这样处理

  1. 从最近任务中彻底关闭 HM互传，然后重新打开一次。首次隐私确认结束后，第二次启动可能会正常完成目录初始化。
  2. 如果出现下载目录或文件访问授权提示，请选择允许，然后再次关闭并打开 HM互传。
  3. 如果没有弹窗，可进入“设置 → 应用和元服务 → 应用管理 → HM互传 → 权限”，查看是否有下载目录或文件访问权限；如有，请允许后重新启动应用。
  4. 确认 Mac 和 MatePad 使用同一个普通 Wi-Fi，而不是访客网络；同时在 Mac 的“隐私与安全性 → 本地网络”中允许 HMTrans。

如果设置中没有相应权限，重新打开后也仍然搜不到：不需要反复卸载。当前 v0.1 市场版无法保证在这种状态下恢复，请等待下个版本更新。

为什么会出现这个问题

应用市场首次启动时会先处理托管隐私确认，HM互传 v0.1 同时开始初始化下载目录。部分设备上目录授权请求没有显示系统弹窗，而是直接返回未授权。

当前版本又把下载目录、TCP 文件接收和 UDP 设备发现串在同一条启动链路上：目录没有准备好，接收服务就无法启动，随后设备发现也会停止，因此两端看起来像是完全搜不到彼此。

v0.2 如何解决

以下修复已经进入 v0.2 发布候选代码；应用市场正式更新前，当前 v0.1 用户仍需使用上面的临时处理办法：

  • 使用 HM互传自己的隐私确认流程，同意前不启动局域网服务，同意后按顺序初始化本地能力。
  • 设备发现与下载目录、TCP 接收服务解耦；目录异常不再导致两端互相搜不到。
  • HarmonyOS 6.1 / API 23+ 按当前系统文件能力初始化 Download/HMTrans，不再把旧式权限申请作为发现服务的启动条件。
  • 补充明确的错误提示、重新初始化和权限恢复入口，不要求用户卸载重装。
  • 显式启用 UDP 广播，并按当前网卡计算广播地址，避免广播失败被静默忽略。
  • 发布前必须再用应用市场干净安装包验证首次启动、拒绝后恢复、切后台返回以及升级安装。

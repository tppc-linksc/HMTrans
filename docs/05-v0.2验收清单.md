# v0.2 验收清单

> 勾选项必须有对应构建、自动化测试、截图或真机记录。2026-07-12 的代码审查结论和未实测项目见《06-v0.2发布前检查报告》；未勾选不等于未实现，也可能表示尚无足够验收证据。

本清单用于开发阶段验收和发布签字。构建成功、安装成功或单次传输成功都不能单独代表 v0.2 完成。

## 1. 版本和正式包

- [ ] macOS v0.2 正式 DMG 安装后可以启动、打开状态栏和传输。
- [ ] HarmonyOS DevEco 调试包完成基础回归。
- [ ] HarmonyOS 正式签名包完成首次启动和后台测试。
- [ ] HarmonyOS 应用市场安装包完成干净安装和升级安装测试。
- [ ] v0.2 双端完整能力通过。
- [ ] v0.2 Mac 与 v0.1 Pad 可进行基础一对一传输。
- [ ] v0.1 Mac 与 v0.2 Pad 可进行基础一对一传输。
- [ ] 旧版对端在 UI 中明确显示基础兼容模式。

## 2. 隐私和首次启动

- [ ] 首次启动展示 HM互传自己的隐私弹窗，不依赖托管隐私确认服务。
- [ ] 不同意前没有 UDP/TCP socket、局域网扫描或设备信息广播。
- [ ] 同意后记录隐私版本和时间，重启不重复询问同一版本。
- [ ] 隐私版本变化后重新确认。
- [ ] 隐私全文可从应用和官网打开，离线时至少能查看内置基础文本。
- [ ] 系统权限只在实际需要时由系统弹窗申请。
- [ ] 拒绝系统权限后有说明和恢复入口，不要求卸载应用。

## 3. 设备发现和服务解耦

- [ ] 同一普通 Wi-Fi 下首次启动后 10 秒内发现对端。
- [ ] 下载目录不可用时仍能发现设备并显示“可发送、暂不可接收”。
- [ ] TCP 端口失败时 UDP 发现仍工作。
- [ ] UDP 失败时 UI 显示原因、端口和重试入口。
- [ ] 手动刷新、重启发现和重启接收均有效。
- [ ] 网络切换后更新本机地址并重新发现。
- [ ] 访客网络或 AP 隔离时给出可理解提示，不无限转圈。
- [ ] 已配对设备优先单播，广播地址来自活动网卡而非硬编码网段列表。

## 4. 配对和设备管理

- [x] 发现未配对设备时不会直接显示为已连接，未配对文件请求会被拒绝。
- [x] 输入错误或过期六位码时不建立信任关系。
- [ ] 首次连接两端显示相同六位码。
- [ ] 任一端拒绝后不建立信任关系。
- [ ] 配对成功后已配对设备可按设置自动接收。
- [ ] 多台设备名称、平台、系统版本和在线状态正确。
- [ ] 身份指纹变化后强制重新配对。
- [ ] 删除设备后清理数据库信任和系统安全存储。
- [ ] 六位短码和身份指纹原值不出现在数据库诊断事件或导出的诊断文本中。
- [x] UDP 与 TCP 默认端口可分别修改、校验、持久化并重启对应服务。
- [x] 配对成功后接收端立即显示完整设备名称和在线状态，不依赖切换页面刷新。
- [x] 关闭连接后在线设备立即转为离线，传输页同步清除发送目标。
- [x] 在线、已配对离线和未配对设备在同一张附近设备卡片中按绿、蓝、白区分。

## 5. 基础传输和完整性

- [ ] Mac → Pad、Pad → Mac 均可发送文件。
- [ ] 1KB、1MB、1GB、10GB、50GB 文件通过。
- [ ] 空文件、中文名、空格、特殊字符和长文件名通过。
- [ ] 文件重名时预约的最终名称稳定，续传不产生新的重复名称。
- [ ] 接收完成后 SHA-256 与源载荷一致。
- [ ] 校验失败不发布最终文件，并提供从头重传入口。
- [ ] 传输期间内存不随文件总大小线性增长。

## 6. 临时文件和公开目录

- [ ] 传输中 `Download/HMTrans` 不出现多个分片。
- [ ] 默认情况下用户只看到成功文件。
- [ ] 后台被杀后私有 `.part` 仍存在并可恢复。
- [ ] 成功发布后内部 `.part` 和不再引用的 Staging 被清理。
- [ ] 删除已完成记录不删除用户文件。
- [ ] 删除未完成任务时明确询问是否删除可恢复数据。
- [ ] 群发共享归档不会因删除一个子任务而提前清理。
- [ ] 低磁盘和磁盘满时给出预计所需空间与清理入口。
- [ ] 10GB/50GB 从私有目录发布到公开目录时记录耗时和峰值磁盘占用。

## 7. 暂停、取消和断点续传

- [ ] 发送端和接收端都可暂停。
- [ ] 暂停后网络停止、任务和 `.part` 保留。
- [ ] 继续后从确认偏移开始，不从头发送。
- [ ] 取消可选择保留或删除部分数据。
- [ ] 10%、50%、95% 断网后恢复。
- [ ] 10%、50%、95% 杀进程后重新打开恢复。
- [ ] 数据库偏移大于磁盘实际长度时能安全校正。
- [ ] 最后完整分块哈希不一致时回退或从头传，不继续写入错误数据。
- [ ] 同一 `transferId` 重复请求具有幂等性。

## 8. 后台和生命周期

- [ ] 应用内部切换设备、任务、历史和设置页面不影响传输。
- [ ] Pad 返回桌面、打开其他应用和短时锁屏后活动传输继续。
- [ ] HarmonyOS 活动传输时存在合规后台通知，结束后长时任务立即停止。
- [ ] 通知暂停和取消与应用内状态一致。
- [ ] Mac 主窗口关闭后状态栏仍能查看并控制任务。
- [ ] Mac 睡眠/唤醒后任务进入等待重连并恢复。
- [ ] 系统强制中断后重新打开，任务状态和断点仍在。
- [ ] 空闲时不使用活动传输后台任务维持全天候高频发现。
- [ ] 页面和窗口反复打开关闭不会重复创建 listener、timer 或后台任务。

## 9. 文件夹语义

- [ ] 用户发送文件夹，接收端最终得到同名文件夹而不是 ZIP。
- [ ] UI 和历史展示原始文件夹名、文件数和来源大小。
- [ ] 内部 ZIP 名和路径不出现在用户界面。
- [ ] 中文、多层、隐藏文件和大量小文件通过。
- [ ] `../`、绝对路径和符号链接越界用例被拒绝。
- [ ] 中断后复用已准备归档，不重复压缩整个目录。
- [ ] 解压失败时不发布半成品目录。

## 10. 多设备和群组

- [ ] 同时选择两台及以上设备发送。
- [ ] 每台设备独立显示进度、速度和结果。
- [ ] 一台拒绝、离线或空间不足不影响其他设备。
- [ ] 群组全部暂停、继续和取消正确。
- [ ] 只重试失败设备，不重复发送给已完成设备。
- [ ] 同一文件夹只准备和计算一次哈希。
- [ ] 并发达到上限后其余任务稳定排队。

## 11. Mac 状态栏体验

- [ ] 自绘模板图标在浅色、深色、高对比度和 Retina 下清晰。
- [ ] 图标空闲时无持续动画和高频刷新。
- [ ] Finder 文件进入状态栏图标热区时轻量面板及时出现。
- [ ] 单台已配对在线设备可以直接放下发送。
- [ ] 多台已配对在线设备可以明确选择单台或发送到全部。
- [ ] 未配对设备不会静默直发，而是进入配对引导。
- [ ] 拖出图标和面板后不会误发送，面板平滑收起。
- [ ] 无拖拽悬浮时显示任务名称和进度。
- [ ] 精简面板暂停、继续、取消和打开任务中心有效。
- [ ] 主窗口隐藏和面板关闭不影响任务。
- [ ] 多显示器、状态栏空间不足和刘海屏场景下位置正确。
- [ ] Dock、主窗口和状态栏拖拽产生相同的持久任务模型。

## 12. 操作、历史和错误

- [ ] 接收服务、自动发现开关和手动刷新可用。
- [ ] 单任务暂停、继续、取消、重试、删除和打开目录可用。
- [ ] 一键清除已完成、失败和全部历史有清晰范围提示。
- [ ] 清空历史不删除已接收文件。
- [ ] 所有失败都有错误码、中文原因、建议操作和复制诊断入口。
- [ ] 端口占用、空间不足、目录不可写、对端离线、版本不兼容和校验失败均有专项提示。
- [ ] 诊断导出不包含文件内容、私钥、密钥或未脱敏敏感路径。

## 13. UI、无障碍和适配

- [ ] Mac 主窗口、状态栏面板适配深浅色和高对比度。
- [ ] Pad 全屏、分屏、小窗没有重叠、裁切和不可点击区域。
- [ ] 文件准备、等待确认、传输、校验、解压和发布均有状态反馈。
- [ ] 状态不能只用颜色表示。
- [ ] 长文件名可辨识且详情可查看完整名称。
- [ ] 键盘、VoiceOver/系统辅助功能可读取主要控件和进度。
- [ ] 缩略图失败回退类型图标，不影响任务列表。

## 14. 性能和能耗

- [ ] Mac 后台空闲 30 分钟记录 CPU、Energy Impact、wakeups 和网络包数量。
- [ ] 优化后没有 0.1 秒 RunLoop 自旋或每秒主线程设备清理。
- [ ] HarmonyOS 空闲时不向大量硬编码网段持续广播。
- [ ] UI 进度更新不超过设计频率，窗口隐藏后停止高频 UI 刷新。
- [ ] 数据库检查点不按每个网络分块提交。
- [ ] 传输完成后后台任务、哈希线程、计时器和临时句柄全部释放。
- [ ] 1GB、10GB、50GB 记录速度、CPU、内存、耗电和磁盘峰值。
- [ ] 大量小文件和文件夹压缩记录 CPU 与内存峰值。
- [ ] 缩略图生成不抢占活动传输资源。
- [ ] Mac 使用 Instruments Energy Log/Time Profiler/Network，HarmonyOS 使用 DevEco Profiler 留存结果。

## 15. 稳定性

- [ ] 连续运行 24 小时无明显内存增长。
- [ ] 连续运行 24 小时日志、数据库和临时目录不会无限增长。
- [ ] 反复网络切换、暂停继续和窗口开关无崩溃。
- [ ] TCP `51888`、UDP `51889` 实际监听和冲突恢复经过验证。
- [ ] 发布阻断问题关闭，非阻断问题有负责人、风险和后续版本计划。

# v0.2 开发计划

更新时间：2026-07-13。本文按当前三页产品和实际线协议维护，不再保留早期四页 UI、传输圈、`hello/ack` 或 v0.1 降级工作包。

## 1. 已完成实现

- [x] 自有隐私确认；同意前不启动网络和目录权限。
- [x] HarmonyOS 公共 `Download/HMTrans` 权限申请、写测和设置页恢复入口。
- [x] UDP 自动发现、定向确认、12 秒在线 TTL、设备 ID/IP 去重。
- [x] 180 秒六位码、随机安装指纹、身份变化撤销信任。
- [x] 文件和嵌套文件夹、1 MiB 分块、SHA-256、断点续传。
- [x] 暂停、继续、取消保留、取消删除、重试和进程重开恢复。
- [x] 多设备独立子任务；Mac 3 并发、Pad 2 并发；共享源引用计数。
- [x] 活动传输后台保护，空闲释放。
- [x] 三入口 UI：连接、历史、设置；Mac 菜单栏面板；Pad 原生沉浸光感 Dock。
- [x] 历史五状态筛选、按设备分组、Mac 右键、Pad 左滑删除。
- [x] UDP/TCP 弹窗修改、诊断复制、下载目录反馈、临时空间安全清理。
- [x] 深浅色语义资源、窄窗口筛选和连接卡片自适应。

## 2. 2026-07-13 代码收口

- [x] TCP 控制行 64 KiB 上限，双端接收连接上限 16。
- [x] 配对和文件元数据强制校验 `HMTrans/0.2.0`、身份和尺寸字段。
- [x] HarmonyOS 页面离开清理配对/裁剪/持久化计时器。
- [x] 双端清理临时空间前等待发送并发门、接收连接和文件句柄空闲。
- [x] ZIP 路径、盘符、`..`、符号链接、目录大小和条目数防护。
- [x] HarmonyOS 大文件复制和目录统计迁移到 taskpool。
- [x] `Error` 转为可读文字，不再把标准错误显示为 `{}`。
- [x] SQLite 增量事务保存、错误日志、schema 版本和诊断保留上限。
- [x] 删除未使用的旧 V02 协议模型和未使用设备组件；Mac Core 协议模型拆文件。
- [x] HarmonyOS 根页面拆分网络与传输控制器，`Index.ets` 降到 1000 行以内。
- [x] Mac 构建版本优先读取环境变量或当前 git tag，构建号默认使用提交计数。
- [x] Mac 状态栏模板图标、300/450 ms 悬浮延迟和多屏定位。
- [x] 网站响应式和无动画偏好；公开文档同步实际实现。
- [x] Mac TCP 发送/读取加入 30 秒操作超时，半开连接不再永久占用线程。
- [x] Mac 端口探测结果加锁，接收临时文件创建失败返回明确错误。
- [x] Mac 配对/接收决策改为有超时的主线程桥接，移除 `DispatchQueue.main.sync` 跨队列同步。
- [x] Pad 页面销毁后丢弃迟到 UI 回调，移除四张从未使用的原型任务表。
- [x] 双端关键导航、设备、开关和任务操作补充原生辅助功能描述。
- [x] Pad 传输状态集中为稳定枚举，统一 URI 文件名解析；Mac 实际进入 SHA-256“正在校验”状态。
- [x] Pad 配对、接收决策和最终校验回执增加超时，零字节文件也进入校验与发布流程。
- [x] Pad 在接受载荷前预检私有 Staging 空间，保留 64 MiB 安全余量。
- [x] 双端监听系统网络变化，切换 Wi-Fi/网卡后重建发现与接收服务。
- [x] Pad 自动恢复延时器按任务管理并在页面退出时全部取消。
- [x] Pad 网络宿主接口按运行时、状态和操作职责拆分；发送入参改为单一选项对象。
- [x] 双端说明性代码注释统一为中文，协议名、API 名和必要技术术语保留原文。

## 4. 发布前真机工作

以下工作不能由代码审查或本机编译替代，保持未完成：

- [ ] 应用市场正式包干净安装、首次隐私同意、目录授权、拒绝后恢复和升级安装。
- [ ] 5 GB、20 GB+ 文件双向传输、暂停/恢复、SHA-256 和磁盘不足。
- [ ] Pad 锁屏/后台/分屏/浮窗；Mac 睡眠/唤醒；传输中网络切换。
- [ ] 三台设备验证 A-B、B-C 已连而 A-C 未连，以及两个直连目标并发不同任务。
- [ ] 深层嵌套文件夹、ZIP64 大文件夹和恶意归档样本矩阵。
- [ ] Pad 浅色/深色、横竖屏、分屏和最小浮窗逐页截图。
- [ ] 双端空闲 30 分钟和持续传输能耗测量。

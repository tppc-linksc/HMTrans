除“当前实现”或“v0.1 已知问题”外，标注为 `v0.2` 的内容均表示目标方案，不能当作已经交付的能力。

## 文档维护规则

- 产品范围变化时，先更新项目说明和产品交互方案。
- 架构、存储、后台或性能策略变化时，更新技术方案。
- 双端字段、消息顺序或错误码变化时，必须先更新传输协议，再改代码。
- 新增任务必须进入开发计划，并在验收清单中有对应验收项。
- README 和双端子项目 README 只保留摘要，详细定义以本目录为准。
- v0.1 已完成功能不再放入 v0.2 开发计划；只在“当前基线”和兼容章节保留必要说明。

## 不在 v0.2 范围内

- 云端中转、云同步和公网传输。
- 用户账号、登录注册和联系人系统。
- 将文件、传输历史或配对关系上传到服务器。
- 为了兼容旧系统而牺牲 HarmonyOS 6.1 / API 23+ 和 macOS 26+ 的原生体验。

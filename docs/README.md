# HMTrans 官网与基础文档

`docs/` 是 GitHub Pages 的发布目录，只保留官网运行文件和需要长期维护的基础说明。当前官网为 [hmt.tppc.top](https://hmt.tppc.top/)。

## 官网文件

| 路径 | 用途 |
| --- | --- |
| `index.html` | 官网首页、功能、版本状态和下载入口 |
| `connection-help.html` | HarmonyOS 旧版连接问题说明 |
| `privacy.html` | 双端隐私政策固定地址 |
| `site-assets/` | 官网实际使用的图标、设备图片和样式 |
| `CNAME` | GitHub Pages 自定义域名 |

## 基础文档

| 文档 | 内容 |
| --- | --- |
| [项目说明.md](项目说明.md) | 产品原则、当前状态、功能边界和平台基线 |
| [版本历史.md](版本历史.md) | v0.1、v0.2 与 v0.3 已完成能力 |
| [ACL权限说明.md](ACL权限说明.md) | HarmonyOS 受限权限、Profile 和应用市场签名说明 |

研发过程中产生的产品方案、技术方案、协议草案、开发计划、验收清单和发布检查统一放在本地 `work/`。该目录被 Git 忽略，不属于官网或公开仓库；对外长期信息必须归并到本目录的基础文档。

## 维护规则

- 官网页面只引用 `site-assets/`，不要重新建立未使用的 `images/` 或 `design/` 目录。
- 已完成旧版本只在 [版本历史](版本历史.md) 中保留长期摘要；未来规划不写入公开版本历史。
- GitHub Release 和应用市场状态必须分别如实维护；本地 HAP 构建不代表应用市场已经上架。
- 隐私政策地址保持为 `https://hmt.tppc.top/privacy.html`，变更数据处理方式时必须同步双端实现和商店说明。
- Mac 当前使用 ad-hoc 签名且未经 Apple notarization；官网和基础文档只说明真实发布状态。

## 固定产品边界

- 默认局域网直连，不提供云端中转、账号、云同步或公网传输。
- 文件和画面不得上传；诊断、历史和配对关系只保存在本机。
- v0.3.0 只做画面投放，不含声音、反向控制、副屏和内置录制。
- 屏幕画面属于敏感内容，公开发布前必须完成身份验证、会话加密、隐私说明和真机安全验收。

HM互传

同一 Wi-Fi 下，Mac 和 MatePad 直接发现、直接发送原文件；MatePad 还可把系统授权的实时画面加密投放到 Mac。不上传云端，不依赖跨生态账号。

为什么做它

让 Mac 和 MatePad 稳定互传原文件，也能本地投屏

01

局域网直传

设备在同一 Wi-Fi 下自动发现，文件在两端之间直接传输，不经过第三方服务器。

02

保留原文件

面向图片、视频、文档和工程文件等场景，避免聊天工具压缩或改写文件。

03

双端原生

macOS 使用 SwiftUI / AppKit，HarmonyOS 使用 ArkTS / ArkUI，优先贴近系统体验。

04

传输可见

当前传输和历史记录展示方向、进度、速度、大小、格式和结果，失败原因可追踪。

05

可恢复与多设备

v0.2 支持暂停、继续、取消、断点恢复、文件夹还原和多个已连接设备独立任务。

06

本地隐私

无账号、无云端；应用自己的隐私确认完成后才启动局域网服务，发送内容只来自用户主动选择。

07

本地加密投屏

MatePad 每次经系统录屏授权后发送 H.264 画面，Mac 使用 VideoToolbox 解码；画面不写入文件或历史。

支持平台

为桌面和 MatePad 工作流设计

HM互传优先服务个人设备之间的高频文件流转与本地演示：Mac 上拖拽或选择文件，MatePad 上接收后继续编辑；需要展示时，由 MatePad 主动把实时画面投放到 Mac。

macOS

菜单栏常驻、主窗口传输列表、Dock 拖拽入口，以及投屏独立观看窗口、画中画和全屏。

HarmonyOS

ArkUI 界面、系统文件选择、下载目录保存、系统录屏授权、画质选择和投屏网络测试。

版本规划

macOS v0.3.0 已发布，HarmonyOS v0.3.0 待应用市场上传

v0.1

一对一可用

自动发现、TCP 传输、文件和文件夹发送、接收确认、历史记录、SHA-256 校验。

v0.2

后台可靠、可控制、可恢复

已完成断点恢复、活动传输后台保护、六位码配对、多设备、任务控制和文件夹还原。

v0.3

MatePad 单向投屏到 Mac

新增系统授权录屏、H.264 硬件编码、加密媒体链路、Mac 硬件解码、双路接收和投屏网络测试。

v0.3.1

远程请求与反向操作

计划增加 Mac 主动请求投屏，以及经 Pad 单独授权的触控、键盘和鼠标反向操作；当前 v0.3.0 不包含这些能力。

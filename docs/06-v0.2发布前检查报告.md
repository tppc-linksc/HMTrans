# HMTrans v0.2 发布前检查报告

更新日期：2026-07-15

本报告只记录公开仓库可以复核的事实，并严格区分代码验证与真机验收。

## 1. 结论

本轮审计报告中可由代码和公开文档解决的问题已经收口，双端当前均可构建，Mac 核心自动化测试全部通过。v0.2 的代码状态可以进入正式包真机验收，但在大文件、后台、三设备和应用市场干净安装证据完成前，仍应称为“发布候选”，不能把本机构建等同于正式发布验收。

## 2. 已完成的代码修复

### 安全与稳定性

- TCP 控制行限制为 64 KiB，单端同时接收连接限制为 16，避免异常对端导致缓冲区或连接数无限增长。
- Mac TCP 发送和读取等待增加 30 秒操作超时；半开对端会取消连接并进入错误/恢复流程，不再永久占用线程。
- Mac 端口探测结果使用锁隔离，临时接收文件创建失败会返回明确错误。
- Mac 配对与接收确认通过有超时的主线程桥接执行，移除跨队列 `DispatchQueue.main.sync`。
- 配对和文件元数据校验应用名、协议版本、身份、整数范围和必填字段。
- 双端在解压前拒绝绝对路径、盘符、路径穿越和符号链接，并限制中央目录范围、条目数、展开大小和压缩比。
- HarmonyOS 页面销毁时清理配对、设备裁剪和持久化计时器。
- HarmonyOS 按任务跟踪自动恢复延时器，取消、完成或页面退出都会清理，不会在页面销毁后重试。
- HarmonyOS 配对回应和接收决策默认 30 秒超时，最终校验回执上限 10 分钟；零字节文件可正常校验、发布和回执。
- HarmonyOS 接收前会检查 Staging 可用空间并保留 64 MiB 安全余量，不再只等待写盘时才发现空间不足。
- 双端新增网络变化监听，Wi-Fi 或网卡切换后会重建发现与接收监听。
- 清理临时空间前先取消任务并等待发送并发门、接收连接和文件句柄空闲。
- 标准错误统一提取可读信息，不再向用户显示空的 `{}`。
- 活动传输连接与诊断事件均有上限；诊断表只保留最近 500 条。
- Pad 发送端改为在 TCP 建连前注册持续存在的有序控制帧收件箱。小文件的最终成功回执即使早于显式等待到达，也会排队保留，不再因切换监听器而停留“准备中”或循环重连。
- Mac 增加应用级单实例锁；从不同路径启动多个同 Bundle ID 副本时，只有一个实例可以启动发现和接收服务，避免同一设备身份对应多个进程和端口占用冲突。
- Mac 安装身份、指纹和配对关系迁移到 `~/Library/Application Support/HMTrans/Config/pairing.json`；正常覆盖升级保留信任，清除应用数据后生成新身份并要求重新配对。
- 双端新增经过设备 ID 与身份指纹校验的解除配对控制帧；任一端删除设备时会立即通知对端撤销信任并同步退出已连接状态。若对端瞬时不可达，删除端会停止发送受信任确认心跳，使对端在设备 TTL 内自动降级，避免长期显示虚假连接。

### 持久化与性能

- Mac SQLite 改为增量事务写入，增加 schema 版本、回滚和 OSLog 错误记录，不再全表删除后重写。
- 双端数据库迁移均删除从未投入读写的原型表，只保留具有实际读写路径的设备、任务和诊断结构。
- HarmonyOS relationalStore 初始化具备幂等保护和 schema 版本。
- HarmonyOS schema 升至 v3，任务按记录增量保存；早期单行快照迁移后删除，用户清空历史不会在重启后恢复旧数据。
- 大文件 Picker 内容复制与目录统计迁移到 taskpool，避免在 ArkUI 主线程递归同步处理。
- Mac 热路径不再反复创建 `DateFormatter`；状态栏悬浮延迟和多屏定位已修正。
- 后台保护仅在活动传输期间持有，设置切换失败会反馈并回滚显示状态。

### 架构与可维护性

- HarmonyOS `Index.ets` 从 1620 行拆到 967 行，设置开关的事务与失败回滚进入独立控制器。
- 网络/配对/在线状态进入 `PadNetworkController`，多目标发送和任务控制进入 `PadTransferController`；两者均低于 600 行。
- Mac 协议模型从 `HMTransCore.swift` 拆到 `WireProtocol.swift`，删除双端未使用的旧 V02 类型和未使用设备组件。
- 关键协议、ZIP 防护、后台引用计数、控制器边界和恢复逻辑已增加说明性注释。
- 说明性代码注释已统一为中文，仅协议、API 和平台固有名称保留英文。
- Pad 网络 Host 已拆分为运行时、响应状态和业务操作三组小接口；TCP 发送改用参数对象，减少位置参数误传风险。
- 双端关键导航、开关、设备卡片、配对倒计时、任务控制和删除操作已补充原生辅助功能描述。
- Pad 传输状态集中为稳定枚举，重复 URI 文件名解析已统一；Mac 发送前 SHA-256 阶段会显示“正在校验”。

## 3. 产品与 UI 基线

当前双端统一为三个一级入口：连接、历史、设置。

- 连接页负责发现、配对、选择直连设备以及发起文件/文件夹发送。
- 历史页同时展示活动和结束任务，按全部、传输中、已完成、失败、已取消筛选，并继续按设备分组。
- 设置页提供接收、发现、自动接收已配对设备、后台保护、下载目录、临时空间、网络端口、诊断和隐私入口。
- Mac 删除操作位于记录或设备组的右键菜单；Pad 使用左滑单条删除和触摸确认。
- Pad Dock 使用原生 `HdsTabs`、`MaterialType.IMMERSIVE`、`MaterialLevel.EXQUISITE`。
- 公开产品、交互、技术和协议文档已按当前三页界面及真实换行 JSON + TCP 载荷协议重写，不再按早期四页方案审核。

## 4. 构建与自动化证据

### macOS

- Release 应用构建：通过。
- 当前发布版本：`v0.2.1`（构建号 `22`）；最终 DMG 已生成并通过包内版本核对。
- Swift Testing：9 项通过，0 失败。
- 断点续传与双设备并发测试使用不重叠的随机端口段，避免 Swift Testing 并行执行时偶发抢占端口。
- 覆盖：嵌套文件夹还原、断线续传与 SHA-256、六位码校验、双端解除配对控制帧、符号链接与异常压缩比 ZIP 拒绝、两个设备同时接收不同文件、半开 TCP 超时退出、接收中断准确回传任务 ID。
- 构建脚本支持环境变量覆盖版本，并在恰好位于 git tag 时自动读取版本；构建号默认取提交计数。
- 当前仅使用 ad-hoc 本地签名。由于没有 Apple Developer Program 账号，不存在 Developer ID 签名或 Apple notarization，本轮没有伪造或修改这些配置。

### HarmonyOS

- HarmonyOS 6.1 / API 23 目标 HAP：构建通过。
- 当前发布候选版本：`0.2.1`（`versionCode=4`）；应用市场尚未提交审核，应上传本轮最新签名 HAP。
- 已核对包内清单：`buildMode=release`、`debug=false`，不是调试包。
- 协议、任务状态与 ZIP 路径穿越 Hypium 测试源已加入 `entry/src/test`。
- 当前命令行工程未暴露 test task，因此不把“测试源存在”写成“真机测试已通过”；需在 DevEco Local Test/目标设备流程执行。
- 编译器仍会对部分系统 API 输出“可能抛出异常”和能力覆盖提示；相关调用已有错误边界或能力检测，当前没有 ArkTS 编译错误。

## 5. 权限、隐私与公开资料

- 同意应用自有隐私弹窗前不启动 UDP/TCP，也不申请下载目录。
- 同意后由系统目录权限流程准备 `Download/HMTrans`；拒绝后发现和发送仍可用，设置页“检查”可重新申请并给出反馈。
- 发送内容先复制到应用私有 Staging，完成文件才发布到用户可见下载目录；用户不会看到 ZIP 或多个分片。
- 隐私政策已按当前使用的局域网、设备信息、文件选择、下载目录、后台任务、诊断与本地数据库行为更新。
- 官网响应式 CSS 已修复，并补充减少动画、键盘跳转、深色控件和真实 Mac 未公证说明。

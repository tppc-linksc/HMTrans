# v0.2 产品与交互方案

## 1. 设计目标

v0.2 的交互目标不是增加更多按钮，而是让用户始终知道：设备是否可用、任务进行到哪里、为什么失败、下一步能做什么。

基本规则：

- 应用内部切换页面、Mac 主窗口关闭、Pad 切到后台时，活动传输不依赖页面生命周期。
- 自动发现保留，但所有关键能力都提供手动开关、刷新、重试和诊断入口。
- 完成文件才出现在公开目录，内部临时文件不暴露给用户。
- 配对完成后减少重复确认，但取消、暂停和错误处理始终由用户控制。

## 2. 信息架构

双端主界面统一为四个区域：

```text
设备
├── 本机服务状态
├── 附近设备
├── 已配对设备
└── 手动刷新、配对和多选

任务
├── 排队/准备中
├── 正在传输
├── 已暂停
├── 等待重连
└── 失败任务

历史
├── 已完成
├── 已取消/失败
├── 搜索、筛选和多选
└── 删除、清空和打开目录

设置
├── 接收服务与自动发现
├── 接收确认方式
├── 下载目录与临时空间
├── 并发数量
├── 已配对设备
├── 隐私政策
└── 诊断信息
```

## 3. Mac 状态栏体验

### 3.1 状态栏图标重绘

当前通用双向箭头 SF Symbol 替换为 HM互传专用模板图标。

设计要求：

- 采用单色矢量模板资源，设置 `isTemplate = true`，自动适配深浅色和高对比度。
- 视觉语义是“Mac/屏幕与 Pad/设备之间双向传输”，但在 16～18pt 视觉尺寸下仍保持简洁。
- 使用像素对齐的 1.5～2pt 主线，不使用渐变、彩色 App 图标或细小文字。
- 至少验证 1x/2x、浅色、深色、选中态和高对比度。
- 状态变化不做持续旋转动画，避免耗电和视觉干扰；活动传输可使用轻微实心变化或小型状态标记。

建议资源状态：

- `StatusIdle`：默认双端轮廓。
- `StatusTransferring`：传输通道加粗或增加小型实心标记。
- `StatusAttention`：任务失败或服务异常，使用系统强调状态，不用高频闪烁。

本次文档只确认设计和实现要求，图标资产在 v0.2 UI 工作包中绘制并真机检查。

### 3.2 拖拽快速发送

用户从 Finder 拖动文件或文件夹到 HM互传状态栏图标时，图标进入高亮，图标下方弹出轻量 `NSPanel`。实现上使用 AppKit `NSStatusItem` 自定义拖放视图和锚定面板，替换当前只提供菜单命令的 `MenuBarExtra`。

交互规则：

1. 文件进入状态栏图标热区后立即打开轻量发送面板。
2. 只有一台在线且已配对设备时，面板显示设备和文件摘要，松手后直接创建任务。
3. 有多台在线且已配对设备时，面板显示设备目标；用户把文件放到单台设备或“发送到全部”区域。
4. 设备未配对时不能静默发送，面板引导打开配对流程。
5. 已配对但离线时不假装发送成功；显示离线状态，可进入主窗口等待设备上线后再操作。
6. 拖出图标和面板热区后延迟约 300～500ms 收起，避免跨越图标与面板间隙时误关。
7. 放下文件后面板切换为任务摘要，短暂停留后折叠；传输继续由全局协调器处理。

拖拽面板展示：

- 文件/文件夹名称和数量。
- 总大小；准备文件夹时显示“正在准备”。
- 在线已配对设备。
- 当前默认目标、单设备目标和“发送到全部”。
- 空间、权限、离线或协议不兼容提示。

### 3.3 悬浮精简任务列表

没有拖动文件时，鼠标在状态栏图标上停留约 300ms，弹出精简任务面板。

- 最多直接展示 3 个活动任务，其余显示“还有 N 个任务”。
- 每项展示文件名、目标设备、方向、百分比和进度条。
- 等待重连、已暂停和失败状态使用文字说明，不能只靠颜色。
- 任务行提供暂停/继续和取消入口；危险操作需要二次确认或撤销机会。
- 底部提供“打开任务中心”和接收服务状态。
- 没有任务时仅显示“暂无传输”、附近在线设备数量和“打开主窗口”，不持续刷新空界面。

点击状态栏图标默认打开同一个精简面板；右键或面板中的更多按钮提供主窗口、选择文件、接收服务、设置和退出。

### 3.4 生命周期

- 状态栏、轻量面板和主窗口共享同一个 `TransferCoordinator`，不得各自创建发现或接收服务。
- 关闭主窗口只隐藏窗口，不取消任务。
- 面板关闭只停止面板 UI 更新，不停止网络任务。
- 拖拽目标解析、文件准备和发送均进入统一任务队列，禁止另外实现一条不可恢复的“快捷发送”链路。

## 4. HarmonyOS 主界面

- 顶部显示本机名称、接收服务、发现服务和下载目录状态。
- 附近设备支持刷新、多选和配对；设备卡片展示完整名称、系统版本、在线/离线和协议能力。
- 任务中心独立于首页，应用切换页面时传输继续。
- Pad 切到后台或锁屏后，活动传输通过系统长时任务继续，并在通知中显示名称、进度、暂停和取消。
- 页面恢复时从数据库重新订阅状态，不重建正在运行的任务。

## 5. 任务控制

单任务支持：暂停、继续、取消、等待重连、失败后继续、从头重传、删除记录、打开所在目录、复制诊断信息。

群组任务支持：全部暂停、全部继续、全部取消、只重试失败设备，并展开查看每台设备的独立进度和结果。

历史记录支持：删除单条、多选删除、清除已完成、清除失败、清除全部和清理临时文件。删除已完成记录不删除用户文件；删除未完成任务时明确询问是否删除可恢复数据。

## 6. 接收与配对

接收策略提供三档：

- 已配对设备自动接收。
- 每次询问。
- 拒绝所有接收。

首次配对显示六位码，两端确认一致后保存设备身份。多台已配对设备同时在线时，状态栏拖拽必须让用户明确选择目标，不能依靠不透明的自动选择。

## 7. 错误和诊断

所有失败都包含错误码、用户可读原因、建议操作和复制诊断入口。任务详情至少展示传输 ID、对端、文件路径、进度、速度、重试次数、协议版本、关键时间和最近事件。

常见恢复入口：重新扫描、重启接收服务、重新检查目录、继续任务、从头重传、打开设置和复制诊断信息。

## 8. 隐私首次启动

首次启动展示 HM互传自己的隐私摘要、隐私政策和同意/不同意按钮。同意前不启动 UDP、TCP、局域网扫描或设备信息广播；同意后再进入目录检查和服务初始化。系统权限如果仍有必要，必须在对应功能触发时使用系统弹窗，不能用自定义弹窗冒充系统授权。

## 9. 无障碍和性能约束

- 状态不能只靠颜色表达；按钮提供可读标签和键盘焦点。
- 长文件名中间截断，但详情页可查看全名。
- UI 进度限频，不随每个网络分块刷新。
- 悬浮面板关闭后停止订阅高频 UI 更新。
- 缩略图在后台低优先级生成，传输活跃时让路。

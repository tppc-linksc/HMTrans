# v0.2 产品与交互方案

本文以 2026-07-13 的双端实际界面为准，替代早期“四个 Tab”“传输圈”和独立任务页方案。

## 1. 信息架构

双端只有三个一级入口：

| 入口 | 目的 | 主要操作 |
| --- | --- | --- |
| 连接 | 建立可信直连并发起发送 | 查看网络、配对、多选目标、选择/拖入文件或文件夹 |
| 历史 | 统一管理当前和过去任务 | 筛选、暂停、继续、取消、重试、删除和清空 |
| 设置 | 管理本机能力和本地数据 | 开关服务、检查目录、清理临时数据、改端口、复制诊断、查看隐私 |

不再设置独立“传输”页。发送区位于连接页底部；活动任务和历史统一放在历史页。

## 2. 连接页

### 2.1 顶部信息

- 当前网络卡片显示 Wi-Fi 图标、真实 SSID 和本机局域网地址；不显示 UDP/TCP 端口。
- 配对码卡片显示六位码和 180 秒环形倒计时。点击倒计时可立即换码。
- 连接页没有手动刷新按钮，UDP 发现自动刷新。
- 接收/发现总开关不在连接页重复出现，由设置页分别控制。

### 2.2 附近设备

同一张“附近设备”区域中展示三种状态，每台物理设备按稳定设备 ID/IP 去重：

- 绿色：双向心跳确认、已配对、当前在线。
- 蓝色：已配对但当前离线；点按重新发现并连接，不重复输入配对码。
- 普通底色：新设备；点按后输入对方当前六位码。

设备卡片展示系统设备名、平台、系统版本和连接状态。Pad 使用短卡片并支持换行排列；Mac 显示类似 `Linksc的MatePad Pro · HarmonyOS6.1.0.117 SP6` 的信息。

在线设备可点按选择/取消发送目标。至少保留一个目标；多目标不会因为新设备上线而被静默扩展。Pad 长按已配对设备、Mac 通过设备菜单可移除本机保存的信任关系，操作前必须确认。

### 2.3 发送区

- Mac：点击“+”选择，或把文件/文件夹拖入虚线区域。
- Pad：点按“+”调用系统混合文件/文件夹选择器；外接鼠标/键盘环境也接受系统拖放。
- 发送区明确显示将发送到的设备名或设备数量。
- 选择多个已连接设备时，每个设备生成独立子任务、独立进度和结果。
- 用户可以先给设备 A 发送文件 1，再选择设备 B 发文件 2；任务可以并行，但产品不提供一次对话框内的“文件到设备”路由矩阵。

## 3. 配对

1. 新设备点按后弹出六位数字输入框。
2. 发起方把输入码、随机安装 ID、随机身份指纹、设备名和接收端口发给对端。
3. 对端只在配对码未过期、完全匹配且身份指纹非空时接受。
4. 双方分别保存对方设备 ID 与指纹；后续相同指纹可自动接收或按设置询问。
5. 指纹发生具体变化时撤销本机信任并要求重新配对；缺少指纹的单个发现包不会误删已有信任。

Pad 显示已连接、Mac 显示未连接时不算成功：只有双方在 12 秒 TTL 内持续收到明确指向本机的确认心跳，卡片才进入绿色在线状态。

## 4. 历史页

### 4.1 统一状态筛选

顶部使用一整块分段控件：`全部 / 传输中 / 已完成 / 失败 / 已取消`。整个分段区域都可点击，窄窗口按可用宽度等分，不留不可点击缝隙。

每个筛选结果仍按设备分组，最近有活动的设备排在最前。任务至少显示文件名、方向、时间、进度/结果、错误摘要和速度或完成信息。

### 4.2 当前任务

- 暂停：关闭或暂停当前网络流，保留私有分片和确认偏移。
- 继续：恢复已有流，或在断连时等待对端重新上线后协商断点。
- 取消并保留：任务进入已取消，保留可恢复源/分片。
- 取消并删除：取消后清理该任务可恢复数据。
- 组操作：同一多设备发送组可整体暂停/继续/取消，各子任务状态仍独立。

### 4.3 已结束任务

- 失败/已取消的发送任务可重试；源文件副本不存在时明确要求重新选择。
- Mac 在记录上右键提供打开、Finder 中显示、重试、删除此记录、清空此设备、清空全部。
- Pad 使用触摸交互：左滑一条记录只露出红色删除区；设备标题提供清空本设备，顶部“清空记录”清空全部。
- 所有清空操作二次确认。删除历史不删除已经发布到下载目录的文件。
- 历史不按数量或时间自动清理。

## 5. 设置页

开关必须真实控制服务并持久化：

- 接收服务：关闭后仍可发现和发送，但不监听 TCP 文件接收。
- 自动发现：关闭后停止 UDP 心跳，设备变为离线；不自动删除配对关系。
- 已配对设备自动接收：关闭后已配对设备每个传输也需确认；陌生设备永远先配对。
- 后台传输保护：只在存在活动任务时申请系统长时任务/防休眠，空闲时释放。

操作项顺序为：诊断信息、下载目录、临时传输空间、网络服务。网络服务行显示当前 UDP/TCP 和运行状态，点按后在弹窗中同时修改两个端口并重启服务。

- “复制诊断”成功或失败必须有即时反馈。
- “下载目录”检查会重新申请被拒权限、验证可写性，并用 Toast 反馈。
- “清除临时传输空间”先取消任务，等待网络和文件句柄退出，再删除私有暂存；不删除成功文件和历史。
- 隐私与本地数据单独成块，链接官网隐私政策。

## 6. 平台交互

### Mac

- 左侧整行导航均可点击，入口为连接、历史、设置。
- 使用系统浅色/深色和材质，不模拟 HarmonyOS 控件。
- 菜单栏使用单色模板图标。悬浮约 300 ms 后显示精简任务面板，离开约 450 ms 后收起；拖拽文件进入图标区域打开发送面板。

### Pad

- 页面背景延伸到系统状态栏，不再放顶部应用标题。
- 底部 Dock 使用 HarmonyOS 原生 `HdsTabs`、`MaterialType.IMMERSIVE` 和 `MaterialLevel.EXQUISITE`；不是手绘彩色背景。
- 全屏、横竖屏、分屏和浮窗均从顶部向下排版。宽度小于 760 vp 时网络/配对卡片上下排列，筛选按钮等分可用宽度。
- 触摸优先：点按、长按、左滑和确认弹窗替代依赖右键/悬浮的小按钮。

## 7. 错误和反馈

- 用户可恢复错误使用中文原因和下一步操作，不直接展示 `{}` 或原始系统对象。
- 任务错误保留稳定模块码，例如发现、接收、校验、空间、归档。
- 诊断报告脱敏 IP 和完整路径，不包含文件内容、配对码或身份指纹原值。
- 关键设置、权限检查、配对、选择器失败和清理操作必须 Toast/按钮状态反馈，不能只写入不可见变量。

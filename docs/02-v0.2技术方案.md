# v0.2 技术方案

## 1. 约束

- HarmonyOS 6.1.0 / API 23+；macOS 26+、Swift 6。
- 文件只在局域网内直传，不实现服务器、中继、账号或云同步。
- v0.2 两端使用同一套换行 JSON 控制帧 + 原始 TCP 文件载荷；不声明尚未实现的 `hello/hello_ack` 协商或 v0.1 降级。
- 六位码是人工配对门禁，随机安装指纹用于后续身份一致性检查；没有 TLS/E2E 内容加密。

## 2. 双端结构

### macOS

- `HMTransCore`：实际线协议模型、TCP 发送/接收、SHA-256、接收结果发布和 UDP 发现。
- `HMTransMacApp`：SwiftUI/AppKit 界面、`TransferViewModel`、菜单栏 `NSStatusItem/NSPanel`、后台活动和 SQLite。
- `TransferViewModel` 通过发现、文件选择、状态扩展拆分；单个源文件均控制在约 1000 行以内。
- 网络阻塞桥接封装在 Core 的专用队列，不在主线程执行文件流。

### HarmonyOS

- `components/`：连接、历史、设置、原生 Dock 和弹层。
- `services/`：发现、TCP、目录、归档、哈希、后台、诊断、偏好、任务状态和多目标协调。
- `persistence/TransferStore.ets`：关系数据库边界。
- `pages/Index.ets`：ArkUI 响应状态与服务编排；网络、存储和归档细节不得回流到组件。

服务通过构造参数和回调显式组合。涉及文件/网络的服务可独立构造；UI 页不直接创建套接字或数据库语句。

## 3. 设备发现与在线定义

- UDP 默认端口 `51889`，TCP 默认端口 `51888`，均可由用户同时修改。
- 两端每 5 秒发送一次发现心跳，设备显示 TTL 为 12 秒。
- 收到发现包后向对端实际地址发送带 `acknowledgedDeviceId` 的定向确认。
- 只有已配对、指纹匹配且最近确认心跳明确指向本机，才视为“已连接”。单向收到广播只表示“发现”。
- 发现列表按稳定 `deviceId` 去重，并以 IP 作为同一物理设备的辅助去重条件；TCP 端口探测只刷新已有设备，不能伪造第二张设备卡片。
- 多网卡场景优先使用真实数据报来源/对端广告的可用局域网地址，拒绝本机地址作为发送目标。

## 4. 配对与信任

- 每端生成随机安装 ID 和随机身份指纹；它们不是硬件序列号或账号。
- 六位码有效期 180 秒，比较时去除空格但必须恰好匹配。
- 接受配对必须同时满足：码有效、请求指纹非空、请求字段可解析。
- 保存 `deviceId -> fingerprint`。收到具体不同指纹时撤销本机信任；字段缺失不等于身份变化。
- 配对关系只保存在各自设备本地。移除设备只删除本机信任，不远程修改对端。
- 安装 ID、身份指纹和信任关系都属于应用数据，不绑定硬件。Pad 保存于应用沙箱偏好；Mac 保存于 `~/Library/Application Support/HMTrans/Config/pairing.json`。
- 正常覆盖安装和 v0.2.x 升级保留上述数据，因此可自动恢复信任；清除应用数据后会生成新安装身份，下次必须重新输入六位码。macOS 仅把 `.app` 移到废纸篓不会自动删除 Application Support，完整重置必须同时清除该目录。
- Mac 使用 `~/Library/Application Support/HMTrans/Runtime/application.lock` 作为进程锁。同一账号下不同路径中的 HMTrans 只能有一个实例启动网络服务，避免两个副本使用同一身份广播却只有一个成功占用 TCP 端口。

## 5. 传输与续传

1. 发送端把用户选择内容复制/归档到应用私有持久 Staging，避免系统 Picker URI 失效。
2. 计算完整载荷 SHA-256，按 1 MiB 分块描述任务。
3. 接收端返回 `resumeOffset`；偏移必须在 `[0,fileSize]` 内。
4. 发送端从确认偏移继续原始 TCP 字节流。
5. 接收端写入 `.part`，完成后校验 SHA-256。
6. 普通文件原子移动到目标；文件夹 ZIP 先验证中央目录、绝对路径、`..`、符号链接、条目数量和目录大小，再解压并发布。
7. 成功后删除私有载荷；失败/断连按任务策略保留，默认最多 7 天。

控制帧单行最大 64 KiB；接收端同时打开的 TCP 连接最多 16，超限立即拒绝。Mac 每次 TCP 发送/读取等待默认 30 秒，Pad 配对回应和接收决策默认 30 秒，最终 SHA-256 校验回执最长等待 10 分钟。超时后主动关闭当前连接并进入可恢复状态，避免错误或恶意对端造成无限缓冲、连接增长或永久占用线程。

Pad 发送端从建立连接开始到最终结果返回始终使用同一个有序控制帧收件箱。`receive_decision` 与 `transfer_success/transfer_failed` 即使在小文件场景连续到达，也会按序排队消费，不能因切换消息监听器丢失最终回执。

接收端在回复接受前检查私有 Staging 可用空间，并在剩余载荷之外保留 64 MiB 安全余量。零字节文件不等待不存在的载荷回调，接收决策送出后立即进入空文件 SHA-256、发布和结果回执。

## 6. 多设备与并发

- 每个目标设备对应独立 `transferId`；一次多选形成共享 `groupId`。
- 同一源归档可在组内引用计数共享，所有子任务完成且不需要恢复后才删除。
- Mac 网络会话并发上限 3，Pad 上限 2；其余任务排队，不共享进度或错误。
- 取消/清理会使当前传输代次失效，排队任务获得并发许可后先检查代次，不会在“清空临时空间”后重新启动。
- 应用不做中继或网状路由。A-C 不直连时，B 不自动转发 A 的文件。

## 7. 本地持久化

### macOS SQLite

- `devices`、`transfers`、`diagnostic_events` 表由 `PRAGMA user_version=2` 标记；未投入读写的原型表在迁移时删除。
- 任务保存采用按 ID 比较后的增量 upsert/delete 和事务，不再每次全表删除重写。
- prepare/step/commit 错误写入 OSLog，并可被 ViewModel 读取反馈；事务任一步失败即回滚。
- 诊断事件只保留最近 500 条。

### HarmonyOS relationalStore

- `devices`、`transfer_records` 和 `diagnostic_events` 使用独立表；每个任务按 `record_id` 增量写入，不因一次进度更新重写整份历史。
- schema 标记 `user_version=3`，初始化幂等；v3 会迁移早期 `app_state` 单行任务快照，迁移成功后删除旧键，避免用户清空记录后旧历史复活。
- 查询列在读取前校验索引；schema 错误向上抛出，单条损坏记录会被隔离并写入本地诊断，不影响其余历史恢复。
- 诊断事件只保留最近 500 条。

数据库只存元数据。文件字节、分片和共享归档只在私有 Staging；成功文件才进入 `Download/HMTrans` 或 Mac 用户选择的下载目录。

## 8. 后台与生命周期

- Pad 仅在活动任务期间调用 `startBackgroundRunning(DATA_TRANSFER)` 并发布可操作通知；任务归零或开关关闭后停止。
- Mac 仅在活动任务期间持有防空闲休眠活动；系统睡眠前把任务标记为等待重连，唤醒后恢复发现。
- 双端监听系统网络变化；Wi-Fi 或活动网卡切换后停止旧监听，重新绑定接收端口并开始发现。
- Core 的同步接收决策通过有超时的主线程桥接执行，不使用 `DispatchQueue.main.sync` 形成跨队列同步环；UI 无响应时按拒绝处理。
- 页面离开时清除配对、设备裁剪和持久化计时器，停止页面拥有的发现/接收服务。
- Pad 每个自动恢复任务都保留可取消的延时器；任务取消、完成或页面退出时立即清理。
- 页面失效后丢弃迟到的 UI 回调，不从网络线程直接修改已销毁的 ArkUI 响应状态。
- 清理临时空间先取消发送/接收，再等待并发门、网络连接和文件句柄空闲；超时则保留数据并提示重试。

## 9. 权限与隐私

- 自有隐私弹窗是服务启动门：同意前不请求目录权限、不广播设备、不监听 TCP。
- Pad 在同意后请求 `READ_WRITE_DOWNLOAD_DIRECTORY`，建立并实际写测 `Download/HMTrans`；拒绝不阻断发现和发送，可在设置重新检查。
- 文件选择器只授予用户主动选择内容；应用立即复制到私有持久目录。
- Mac 读取 Wi-Fi SSID 可能触发系统位置授权；拒绝后仍可用局域网地址发现和传输。
- 诊断复制只写剪贴板，不读取剪贴板；报告去除完整本地路径和 IP。

## 10. 性能与安全

- 哈希、大文件复制和目录统计离开 ArkUI 主线程；归档仍使用系统能力。
- 进度持久化 750 ms 防抖，发现 5 秒心跳、4 秒裁剪，空闲无 100 ms 高频轮询。
- 控制帧、连接数、ZIP 中央目录、条目数、展开大小和压缩比均有上限；接收连接 60 秒无有效进度会主动退出。
- 接收文件名去除路径分隔符和平台非法字符；成功目标重名时生成唯一名称。
- 双端关键导航、开关、设备、任务控制和破坏性操作提供原生辅助功能文本；最终朗读顺序仍按真机清单验证。
- 代码不记录文件内容、配对码、指纹原值或签名密码。

## 11. 构建与签名事实

- HarmonyOS 使用 DevEco/hvigor 构建；HDC 覆盖安装要求设备现有包与待安装包签名一致。
- Mac 构建脚本支持 `HMT_VERSION`、`HMT_BUILD_NUMBER` 覆盖版本，默认生成 ad-hoc 签名 App。
- 当前没有 Apple Developer Program 账号，因此没有 Developer ID 签名和 Apple notarization；不把本地 `codesign -` 当作正式公证。

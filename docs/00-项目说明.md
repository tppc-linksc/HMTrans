# 项目说明

HMTrans（中文名：HM互传）是一个面向 Mac 和 HarmonyOS MatePad 的局域网原文件互传工具。

目标是在同一 Wi-Fi 下，让两台设备像 AirDrop 一样互相发现、互相发送文件，但不依赖苹果生态、华为生态或云服务。

## 要解决的问题

现有方案的问题：

- AirDrop 不能直接给 MatePad 使用。
- 华为 Share 在 Mac 端体验不稳定。
- 微信、QQ、钉钉、飞书可能压缩或改写文件。
- 网盘需要上传公网，速度和隐私都不理想。
- 第三方工具可用，但不够贴合自己的双端体验。

HMTrans 只做一件事：

同一 Wi-Fi 下，Mac 和 MatePad 之间稳定、快速、无损地传输原始文件。

## 系统适配基线

- HarmonyOS 端以 HarmonyOS 6.1.0 / API 23 及以上为目标，不再按 HarmonyOS 5.0 的最低能力设计交互和能力边界。
- HarmonyOS 端优先使用 ArkTS / ArkUI / HarmonyOS 原生能力，包括窗口化/分屏适配、拖拽、沉浸式界面、系统文件能力、后台任务和新系统视觉特性。
- macOS 端以 macOS 26 及以上为目标，优先使用 SwiftUI、AppKit、菜单栏、Dock 拖拽、系统材质、深浅色模式和新系统原生能力。
- 兼容旧系统不是当前优先级；如果某个新系统 API 会明显改善体验，优先按新系统实现，再在文档中说明最低系统要求。

当前处于 `v0.1` 发版准备阶段。Mac 端通过 GitHub Release 分发 `HMTrans.dmg`；HarmonyOS 端通过应用市场分发，当前仅支持 Pad 端。

`v0.1` 做：

- Mac / HarmonyOS 双端原生应用。
- UDP 局域网自动发现。
- 一对一 TCP 文件传输。
- 文件和文件夹传输；文件夹以 zip 包传输。
- 接收确认和已信任设备自动接收。
- 当前传输、历史记录、速度、大小、格式、方向展示。
- 双端基础常驻接收和断线重连验证。
- SHA-256 校验。

`v0.1` 不做：

- 一对多发送。
- 群组传输。
- 配对码。
- 断点续传。
- 后台传输保证；`v0.1` 只做基础保活和恢复验证，不承诺后台/锁屏时传输一定不中断。
- 公网传输。

## v0.2 待做

- 一对多连接和群组发送。
- 文件夹语义传输：用户发送文件夹，接收端最终得到文件夹。
- 断点续传，传输中支持暂停和停止。
- 已连接设备再次连接时恢复相关传输记录。
- 读取并展示完整设备名称、平台和系统版本。
- 配对码确认。
- 失败文件 UI 和续传入口。
- 传输记录支持左滑删除单条记录，并支持清空全部记录。
- 后台保活和后台传输保证：传输过程中退到后台或短时间锁屏不应中断；系统强制中断后可恢复。

## 双端形态

Mac 端：

- Swift / SwiftUI。
- TCP / UDP socket。
- SHA-256 校验。
- 菜单栏 App + 主窗口。

HarmonyOS 端：

- ArkTS。
- ArkUI。
- TCP / UDP socket。
- 系统文件选择器和拖拽。
- 应用沙箱目录保存接收文件。

共享部分：

- 局域网设备发现协议。
- 文件传输协议。
- 消息字段定义。
- SHA-256 校验规则。
- 信任设备规则。

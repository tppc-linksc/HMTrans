# 项目说明

HMTrans（中文名：HM互传）是面向 macOS 与 HarmonyOS MatePad 的局域网原文件互传工具。两端在同一 Wi-Fi 下直接发现、配对和传输，不经过云端。

## 产品目标

HMTrans 只解决一件事：让 Mac 和 MatePad 之间的原文件传输稳定、快速、无损、可恢复，并尽量少打断用户当前工作。

核心原则：

- 纯本地：不做云端中转、云同步、账号或登录系统。
- 高速度：文件流不经过数据库，不为了压缩率牺牲局域网传输速度。
- 高质量：完成后通过 SHA-256 校验，失败、断连和恢复状态可追踪。
- 低打扰：Mac 主窗口关闭后仍可从状态栏快速发送和查看进度。
- 用户可控：发现、接收、暂停、继续、取消、重试和清理均有明确入口。
- 隐私可控：隐私确认由应用自己的流程完成，同意前不启动局域网服务。
- 轻量：优先使用系统框架，不引入云 SDK、账号 SDK 或重量级第三方组件。

## 版本状态

- `v0.1` 已分发：Mac 端通过 GitHub Release 分发 `HMTrans.dmg`，HarmonyOS 端已通过应用市场分发，当前仅支持 Pad。
- `v0.2` 是当前开发目标：重点从“一对一能传”升级到“后台可靠、可控制、可恢复、可诊断、多设备可用”。
- v0.1 已完成内容不再进入开发计划；v0.2 仍需兼容 v0.1 的基础一对一传输，便于双端错峰升级。

## v0.2 范围

### 可靠传输

- 任务数据库、持久化检查点和应用私有 Staging 存储。
- 暂停、继续、取消、重试、断点续传和进程重启恢复。
- 文件夹语义传输，接收端得到文件夹，不暴露内部归档文件。
- 一对多和群组发送，每台设备独立进度、结果和重试。
- 后台传输保证；系统强制中断时持久化状态，重新打开后恢复。
- 完成文件才发布到 `Download/HMTrans`，未完成分片默认对用户不可见。

### 设备与安全

- 设备发现、下载目录和 TCP 接收服务解耦。
- 动态子网广播、已配对设备单播和可诊断的网络状态。
- 六位配对码和设备身份指纹。
- 完整设备名、平台、系统版本、协议版本和能力信息。
- 已配对设备记录仅保存在本机，身份密钥进入系统安全存储。

### 操作与界面

- 设备、任务、历史和设置四个清晰入口。
- 每个任务可暂停、继续、取消、重试、删除和查看错误详情。
- 一键清除已完成/失败记录和过期临时数据。
- Mac 状态栏图标重新设计为原生模板图标。
- 文件拖到状态栏图标附近时弹出轻量发送面板；已配对设备可快速发送。
- 鼠标悬浮状态栏图标时展示精简任务列表、文件名和进度。
- 常见文件图标和异步缩略图。

### 隐私与性能

- 使用 HM互传自己的首次启动隐私弹窗和隐私政策页面，不再依赖托管隐私确认流程。
- 没有活动传输时不启动后台长时任务，发现服务采用事件驱动和自适应频率。
- 进度、数据库检查点和后台通知限频，避免空闲时高频唤醒 CPU。
- 大文件、锁屏、后台、低磁盘、断网和正式市场包全部纳入验收。

## 不在 v0.2 范围内

- 云端中转、公网传输和远程下载。
- 用户账号、登录、联系人和跨设备云同步。
- 将文件、历史、诊断日志或配对关系上传到服务器。
- TLS 或内容端到端加密；v0.2 先完成设备身份确认，内容加密后续单独评估。

## 系统适配基线

- HarmonyOS 6.1.0 / API 23 及以上，优先使用 ArkTS、ArkUI、系统文件能力和 `dataTransfer` 后台能力。
- macOS 26 及以上，优先使用 SwiftUI、AppKit、`NSStatusItem`、`NSPanel`、系统材质和深浅色模式。
- 兼容旧系统不是当前优先级。

## 双端形态

Mac 端是菜单栏常驻应用加主窗口；HarmonyOS 端是面向 Pad 全屏、分屏和小窗的原生应用。双端共享发现协议、传输协议、状态机、错误码、SHA-256 规则和配对语义，但分别使用系统原生数据库和安全存储实现。

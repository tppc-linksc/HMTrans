# 项目说明

HMTrans（HM互传）是面向 macOS 与 HarmonyOS MatePad 的开源局域网原文件互传工具。两端在同一 Wi-Fi 下直接发现、配对和传输，不经过 HMTrans 云端，也没有账号系统。

## 产品原则

- **纯本地**：UDP 只做同网段发现，TCP 直接传文件；不提供云端中转。
- **速度和质量优先**：原文件字节直传，完成后进行 SHA-256 校验。
- **任务可恢复**：应用私有目录保存未完成载荷、断点和任务元数据；只有成功结果进入用户可见下载目录。
- **用户可控**：发现、接收、目标选择、暂停、继续、取消、重试、逐条删除、按设备清空和全部清空都有入口。
- **空闲低耗电**：后台长时任务仅在活动传输期间开启；发现心跳和界面刷新限频。
- **轻量开源**：优先使用 ArkTS/ArkUI、SwiftUI/AppKit 和系统数据库，不引入账号或云 SDK。

## 当前产品形态

双端都采用三个一级入口：

1. **连接**：显示当前 Wi-Fi、180 秒六位配对码、附近设备三态和文件/文件夹发送区。已连接设备可以多选；发送任务按设备独立执行。
2. **历史**：统一显示正在传输和已结束任务，提供“全部、传输中、已完成、失败、已取消”五种筛选，并按设备分组。
3. **设置**：管理接收、发现、已配对自动接收、后台传输保护、诊断、下载目录、临时传输空间和 UDP/TCP 端口。

Mac 还有菜单栏常驻入口：悬浮时延迟显示精简任务面板，拖入文件时打开轻量发送面板。Pad 使用 HarmonyOS 原生 `HdsTabs` 浮动 Dock，材质为 `IMMERSIVE + EXQUISITE`。

## v0.2 能力边界

- 文件和嵌套文件夹；文件夹内部使用私有 ZIP 载荷，接收端校验路径后还原成原文件夹。
- 1 MiB 分块、SHA-256、暂停/继续/取消、断线等待和同任务 ID 续传。
- Mac 最多 3 个独立网络会话并发，Pad 最多 2 个；同一文件可发送给多个直连设备，也可在已有任务运行时为另一设备选择不同文件。
- 不充当中继。若 A 与 B、B 与 C 已连接而 A 与 C 未连接，A 不能借 B 自动把文件转给 C；需要 A-C 直连或由用户在 B 上发起新的传输。
- 六位码和随机安装指纹用于确认同一局域网内的对端。当前协议没有 TLS/E2E 内容加密，不对抗已控制局域网的主动中间人。

## 系统基线

- HarmonyOS 6.1.0 / API 23+，Pad，全屏/横竖屏/分屏/浮窗。
- macOS 26+，Swift 6、SwiftUI/AppKit、菜单栏和系统材质。
- Mac 构建脚本生成 ad-hoc 签名 `.app`/`.dmg`，当前没有 Developer ID 和 Apple notarization。

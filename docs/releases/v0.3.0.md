# HMTrans v0.3.0

`v0.3.0` 在稳定文件互传基础上新增 MatePad 到 Mac 的本地加密投屏，并继续保持无账号、无云端、局域网直连。

## 主要更新

- MatePad 使用 HarmonyOS 系统屏幕采集授权和原生 H.264 硬件编码，Mac 使用 VideoToolbox 解码并自动打开独立观看窗口。
- 支持普通窗口、画中画、全屏、方向变化、会话统计、停止和有限重连。
- 提供 720P 级、1080P 级、2K 级与 30P/60P 组合，实际可用档位以设备硬件编码能力为准。
- 一台 Mac 最多同时接收两台 MatePad；双路基线限制为不高于 1080P 级/30P。
- Pad 设置页提供投屏网络测试，展示吞吐量、往返延迟、抖动和推荐档位。
- 配对升级为六位码认证的 X25519 临时协商；文件元数据、解除配对和投屏媒体使用用途隔离密钥认证。
- 修复设备名称、IP 字节序、配对重连、解除配对、历史聚合、传输状态、下载目录与多项深浅色交互问题。

## 系统要求

- macOS 26 或更高版本。
- HarmonyOS 6.1.0 / API 23 或更高版本。
- 两台设备位于同一局域网，并完成六位配对码配对。

## 发布与安装

- Mac 版通过 GitHub Release 提供 `HMTrans.dmg`。当前使用 ad-hoc 签名、未经 Apple 公证，首次打开可能需要右键“打开”或在“隐私与安全性”中允许。
- HarmonyOS v0.3.0 已完成当前目标设备测试，仍需使用应用市场正式 Profile/证书签名并上传；GitHub Release 不提供可替代应用市场正式签名的 HAP。

## 当前边界

- 投屏只从 MatePad 发起，Mac 主窗口不提供主动启动入口。
- 不传输声音，不把 MatePad 作为扩展显示器，不在 HMTrans 内录制画面。
- 不提供键盘、鼠标或触控反向操作；重新设计的 Mac 启动入口和反向操作计划放入 `v0.3.1`。
- 投屏画面仅在内存中加密传输和解码显示，不写入文件历史或下载目录。

# v0.2 ArkTS 实现说明

本文只记录 HarmonyOS 端实现入口。产品、协议和验收以仓库根目录 `docs/` 为准。

## 1. 目标结构

```text
entry/src/main/ets/
├── pages/                 设备、任务、历史、设置
├── components/            可复用界面
├── coordinator/           TransferCoordinator
├── discovery/             UDP、在线状态、网络变化
├── transfer/              发送、接收、续传、并发会话
├── persistence/           relationalStore 和迁移
├── storage/               Staging、.part、发布和清理
├── background/            dataTransfer 长时任务和通知
├── pairing/               配对、身份和安全存储
├── diagnostics/           错误码、结构化事件和导出
└── protocol/              v0.1 适配器、v0.2 消息模型
```

页面不得直接拥有 socket、数据库连接或后台任务。页面只订阅协调器状态并发送用户意图。

## 2. 启动顺序

1. 加载本地隐私政策版本。
2. 未同意时展示 HM互传自己的隐私弹窗，不启动局域网服务。
3. 同意后加载 relationalStore、设置和未完成任务。
4. 独立启动 UDP 发现。
5. 独立检查公开下载目录。
6. 目录可用时启动 TCP 接收器；失败时仍保留发现和发送能力。
7. 恢复未完成任务为暂停或等待重连。

旧式下载目录权限不能继续作为发现服务的硬门槛。系统确实需要权限时，在对应功能触发后使用系统弹窗。

## 3. 存储

- 未完成接收、文件夹归档和发送 Staging 放在 `filesDir/HMTrans/TransferData`，不能使用 `cacheDir`。
- 用户只在 `Download/HMTrans` 看到校验和发布成功的最终文件或文件夹。
- 每个任务只有一个 `payload.part`，不产生多个可见分片。
- 数据库只保存元数据、路径和检查点，不保存文件字节。
- Picker URI 无法跨重启持续访问时，发送前复制到持久 Staging。

## 4. 后台

- 模块声明 `dataTransfer` 后台模式。
- 仅在存在活动传输时启动长时任务和通知。
- 通知展示任务名称、进度以及暂停/取消操作。
- 系统准备挂起、网络断开和任务暂停时立即保存检查点。
- 队列为空后停止后台任务，不用长时任务维持空闲发现。

## 5. 性能

- 删除每秒设备列表轮询，改为事件驱动或低频容差定时器。
- 广播地址根据活动网卡计算，不每 5 秒遍历大量硬编码网段。
- UI 进度限频，窗口不可见时停止高频渲染。
- 数据库每秒或每 4～8MiB 提交检查点。
- 文件流使用固定缓冲区，SHA-256 边传边算。
- 压缩、哈希和缩略图使用后台低优先级任务，并在传输时控制并发。

## 6. 开发约束

- 新代码添加必要注释，续传、状态机、后台和配对核心逻辑写详细注释。
- 协议字段变化先更新 `docs/03-传输协议-v0.2.md`。
- 不能以模拟器或调试包代替正式签名和应用市场安装包验收。
- 不能把云端、账号或自动上传诊断信息引入 v0.2。
